# 任务质量评估

## 评估维度

| 维度 | 检查项 | 权重 |
|------|--------|------|
| 完整性 | 所有需求是否都已实现 | 高 |
| 正确性 | 实现是否符合预期逻辑 | 高 |
| 一致性 | 风格/格式/命名是否统一 | 中 |
| 可维护性 | 代码/文档是否易于后续修改 | 中 |
| 文档同步 | 相关文档是否已更新 | 中 |

## 评估流程

### 1. 回顾原始需求
- 查找用户最初的请求
- 理解任务目标和验收标准
- 识别隐含需求

### 2. 逐项检查
- 对照需求列表验证实现
- 检查边界情况处理
- 验证错误处理

### 3. 识别问题
- 标记遗漏的功能
- 标记实现错误
- 标记风格不一致

### 4. 生成报告
- 输出结构化评估结果
- 提供改进建议
- 给出优先级排序

## 评估报告模板

```markdown
## 任务评估报告

### 任务概述
- **任务描述**：[简述任务内容]
- **完成时间**：[时间]
- **涉及文件**：[文件列表]

### 总体评分

| 维度 | 评分 | 说明 |
|------|------|------|
| 完整性 | ✅/⚠️/❌ | [说明] |
| 正确性 | ✅/⚠️/❌ | [说明] |
| 一致性 | ✅/⚠️/❌ | [说明] |
| 可维护性 | ✅/⚠️/❌ | [说明] |
| 文档同步 | ✅/⚠️/❌ | [说明] |

**总体结论**：✅ 通过 / ⚠️ 需改进 / ❌ 未通过

### 需求完成情况

- [x] 需求1：xxx
- [x] 需求2：xxx
- [ ] 需求3：xxx（问题：xxx）

### 发现的问题

1. 🔴 **严重**：[问题描述]
   - 位置：[文件:行号]
   - 影响：[影响范围]
   - 建议：[修复建议]

2. 🟡 **建议**：[问题描述]
   - 位置：[文件:行号]
   - 建议：[改进建议]

3. 🟢 **优化**：[可选改进]
   - 建议：[优化建议]

### 改进建议

1. [具体可执行的建议1]
2. [具体可执行的建议2]

### 后续行动

- [ ] [需要执行的修复动作1]
- [ ] [需要执行的修复动作2]
```

## 评分标准

### ✅ 通过
- 所有核心需求已实现
- 无严重问题
- 文档已同步

### ⚠️ 需改进
- 核心需求已实现，但有遗漏或小问题
- 存在中等问题需要修复
- 文档部分同步

### ❌ 未通过
- 核心需求未实现
- 存在严重问题
- 实现与需求不符

## 问题分级

| 级别 | 标记 | 定义 | 处理 |
|------|------|------|------|
| 严重 | 🔴 | 功能无法使用或有重大缺陷 | 必须修复 |
| 建议 | 🟡 | 功能可用但有改进空间 | 建议修复 |
| 优化 | 🟢 | 可选的增强或优化 | 可选处理 |

## 特定检查项

### 代码变更评估
- 语法正确性
- 逻辑完整性
- 错误处理
- 命名规范

### 文档变更评估
- 版本号更新
- 章节完整性
- 格式一致性
- 链接有效性

### Preview 页面评估
- 功能可用性
- UI 一致性
- 响应式处理
- 与文档描述一致
